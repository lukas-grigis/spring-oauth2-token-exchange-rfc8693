# OAuth 2.0 Token Exchange (RFC 8693)

A Spring Boot demo that shows how OAuth 2.0 Token Exchange works. A Spring Cloud Gateway sits between frontend clients
and backend microservices, swapping user tokens for service-specific JWTs on the fly.

Three frontend demos (Angular, Vue, React) let you see the exchange happening. Log in as a test user, hit an endpoint,
and watch how the token changes between what the frontend sent and what the service received.

## What's inside

```
Frontend (Angular / Vue / React)
  → sends access token to Gateway (:8000)
    → Gateway exchanges token via Keycloak (RFC 8693)
      → Talk Service (:8100) — requires SPEAKER role
      → Review Service (:8200) — requires REVIEWER role
```

The backend is a Spring Cloud Gateway with TokenRelay and RFC 8693 token exchange, two Spring MVC resource servers (Talk
Service, Review Service) with JWT validation, an Eureka service registry, and Keycloak as the auth provider.

The frontends are three identical demo UIs, each built with their framework's conventions:

- Angular 21 (signals, standalone components, ng-openapi-gen)
- Vue 3.5 (Composition API, vue-router, @hey-api/openapi-ts)
- React 19 (context/hooks, react-router 7, @hey-api/openapi-ts)

Distributed tracing is handled by the OpenTelemetry Java agent (zero code instrumentation). An OTel Collector forwards
traces, metrics, and logs to Grafana's LGTM stack. You can trace a request from Traefik through the Gateway into
Keycloak and the downstream service.

## Quick start

You need [Docker](https://docs.docker.com/get-docker/) and [mise](https://mise.jdx.dev/) installed. mise handles Java,
Maven, and Node versions automatically.

```bash
git clone https://github.com/lukas-grigis/spring-oauth-token-exchange-rfc8693.git
cd spring-oauth-token-exchange-rfc8693
mise run demo
```

That's it. The `demo` task builds the project, starts Docker infrastructure (Keycloak, PostgreSQL, Traefik, OTel
Collector, Grafana), sets up the Keycloak realm, starts all four backend services, and launches all three frontends.
Takes a few minutes on first run.

When it's done, open any frontend:

- Angular: http://localhost:4200
- Vue: http://localhost:5173
- React: http://localhost:5174

Press Ctrl+C to stop everything (frontends, services, Docker).

## Test users

| User  | Password | Role     | Can access     |
|-------|----------|----------|----------------|
| alice | alice    | SPEAKER  | Talk Service   |
| bob   | bob      | REVIEWER | Review Service |

Log in as Alice, call the Talk Service mirror endpoint, and you'll see the token exchange in the diff table. It
highlights exactly which JWT claims changed. Try the Review Service permission check to see access denied.

Switch to Bob and the permissions flip.

## What to look for

The mirror endpoints return the HTTP headers the service received. The Authorization header contains a different JWT
than what the frontend sent. The diff table shows what changed field by field: audience, client, timestamps.

Permission check endpoints return 204 (granted) or 403 (denied) depending on the user's role. Alice can access Talk
Service but not Review Service. Bob is the opposite.

Open Grafana at http://grafana.localhost, go to Tempo, and search for traces. You'll see the full request flow from
Traefik through the Gateway into the service, including the token exchange call to Keycloak.

## Service logs

Backend services run in the background. Logs go to `.logs/`:

```bash
tail -f .logs/gateway.log
tail -f .logs/talk-service.log
```

## Observability

Grafana is at `http://grafana.localhost` with Tempo for traces, Loki for logs, and Mimir for metrics.

## OpenAPI

API docs are generated by Springdoc:

- Gateway: http://localhost:8000/swagger-ui
- Talk Service: http://localhost:8000/talk-service/swagger-ui
- Review Service: http://localhost:8000/review-service/swagger-ui

Frontends can generate typed API clients from these specs with `npm run api:generate` (backend needs to be running).
